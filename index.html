Foundation

The Pillars of our Security

Svashasan OS enforces mathematically verifiable security constraints across every component from silicon to the cloud.

Hardware Root of Trust

System boot sequences are anchored securely in silicon. Firmware and OS images are cryptographically verified through integrated TPMs before any code executes, preventing rootkits entirely.

Secure OTA Updates

Fleet-wide patching via signed and authenticated Over-The-Air updates. Advanced A/B partition logic guarantees automatic rollback functionality if an update gets interrupted or fails validation.

Critical patches deployed within 24 hours
High-severity issues resolved within 7 days

Real-time Threat Detection

Heuristic and AI-driven kernel space agents actively monitor system call behavior and network packets. Anomalous behavior is dynamically isolated from critical actuator systems instantly.

Data Privacy & Telemetry

Video logs and autonomous telemetry are stripped of PII locally, heavily encrypted via TLS 1.3, and streamed to the cloud without exposing passenger identities. Compliant with emerging global frameworks.

A Defense-in-Depth Topology

Protecting an autonomous vehicle requires isolating non-critical features from the core driving engine. Our architecture divides the vehicle into strict trust zones.

  • In-Vehicle Network Segmentation

    Infotainment, V2X, and core ADAS compute are segmented across isolated subnets. We enforce strict unidirectional data flow to actuator controls.

  • V2X Communication Security

    All infrastructure interactions use certificate-based mutual authentication to prevent spoofing or man-in-the-middle attacks on traffic signals.

  • Trusted Cloud Environment

    Fleet telemetry is stored in isolated tenant spaces leveraging HSMs (Hardware Security Modules) for key wrapping and encryption-at-rest.

User Privacy

Privacy by Design

Every data decision in Svashasan OS defaults to protecting the vehicle owner. Your data stays yours.

Local-Only Sentry Footage

Footage from Guardian/Sentry Mode is stored locally on-device only and is never uploaded to Svashasan servers unless explicitly shared by the user.

72-Hour Auto-Expiry

Data retention is limited to 72 hours by default for all onboard recordings, after which it is automatically overwritten.

Full Data Control

You can export or permanently delete all your data at any time from the Svashasan app under Settings > Privacy.

Access Control

PIN to Drive

Before the vehicle can be driven, the owner must enter a secure personal identification number. This prevents unauthorized use even if someone has physical access to the key or key card.

How it works
If 5 consecutive incorrect PIN attempts are made, the vehicle enters a temporary lockout for 10 minutes. After 3 lockouts, a verification link is sent to your registered email and phone number to reset access.
